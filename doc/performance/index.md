## 性能优化建议

### 索引优化

1. **增加线程数**：`indexing_threads: 8`（建议 CPU 核心数的 1-2 倍）
2. **排除无关目录**：
   ```yaml
   exclude_dirs:
     - "**/target/**"
     - "**/build/**"
     - "**/node_modules/**"
   ```
3. **限制文件大小**：`max_file_size_kb: 256`

### 搜索优化

1. **使用精确搜索**：避免不必要的 `*` 通配符
2. **限制返回数量**：合理设置 `limit` 参数
3. **按类型过滤**：使用 `config_type`、`dep_type` 等过滤器

### 存储优化

1. **定期清理**：删除不需要的项目索引
2. **WAL 模式**：默认启用，提升并发性能
3. **索引优化**：数据库自动维护索引

## 性能测试方法

运行性能基准测试：

```bash
# 运行所有性能测试
mvn test -Dtest=PerformanceBenchmarkTest

# 运行特定测试
mvn test -Dtest=PerformanceBenchmarkTest#benchmarkIndexing50Files
```

## 已知限制

1. **单写入者**：SQLite 同时只支持一个写入操作
2. **大文件**：超过 `max_file_size_kb` 的文件会被跳过
3. **符号链接**：默认不跟随，需要配置 `follow_symlinks: true`
