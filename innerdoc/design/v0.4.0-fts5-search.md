# v0.4.0 设计文档 — FTS5 全文搜索

## 背景

当前搜索使用 `LIKE '%query%'`，在大代码库（1000+ 文件）中执行全表扫描，性能差。

## 方案

使用 SQLite FTS5 虚拟表实现全文搜索。

### 数据库设计

```sql
-- FTS5 虚拟表（符号）
CREATE VIRTUAL TABLE IF NOT EXISTS symbols_fts USING fts5(
    name, qualified_name, signature,
    content=symbols, content_rowid=id
);

-- FTS5 虚拟表（代码块）
CREATE VIRTUAL TABLE IF NOT EXISTS chunks_fts USING fts5(
    name, content, package_name, class_name,
    content=chunks, content_rowid=id
);
```

### 索引策略

1. 索引时同步写入 FTS 表
2. 使用 SHA-1 增量更新 FTS 表
3. 删除时同步清理 FTS 表

### 搜索逻辑

```java
// FTS5 搜索
String sql = "SELECT rowid FROM symbols_fts WHERE symbols_fts MATCH ?";
// 支持布尔操作：AND, OR, NOT
// 支持前缀搜索：term*
// 支持短语搜索："term1 term2"
```

### 性能对比

| 操作 | LIKE (当前) | FTS5 (新) |
|------|-------------|-----------|
| 1000 文件搜索 | ~50ms | ~5ms |
| 10000 文件搜索 | ~500ms | ~10ms |
