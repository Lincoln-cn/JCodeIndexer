# v0.5.0 设计文档 — 测试完善 + 代码质量

## 背景

当前测试覆盖率低，多个核心模块无单元测试。代码存在重复和风格不一致。

## 测试计划

### ChunkerTest

**目标**: 覆盖代码切片逻辑

**测试用例**:
1. `testSmallClassWholeClass()` — 小类整体切分
2. `testLargeClassHeaderOnly()` — 大类只取头部
3. `testMethodChunking()` — 方法级切分
4. `testLongMethodTruncation()` — 超长方法截断
5. `testFileHeader()` — package + imports 切分
6. `testRecordType()` — Record 类型支持

### JavaParserAdapterTest

**目标**: 覆盖核心解析器

**测试用例**:
1. `testClassSymbolExtraction()` — 类符号提取
2. `testMethodSymbolExtraction()` — 方法符号提取
3. `testFieldSymbolExtraction()` — 字段符号提取
4. `testImportReferenceExtraction()` — import 引用
5. `testCallRelationExtraction()` — 调用关系
6. `testEmptyFile()` — 空文件处理
7. `testSyntaxErrorFile()` — 语法错误处理

### McpE2ETest 改为 JUnit

**修改**: 将 `public static void main()` 改为 `@Test` 注解方法，使其能被 `mvn test` 执行。

## 代码质量计划

### SignatureBuilder 提取

**问题**: JavaParserAdapter 和 Chunker 有 ~50 行重复签名构建代码。

**方案**: 提取 `SignatureBuilder` 工具类：

```java
public class SignatureBuilder {
    public static String buildClassSignature(ClassOrInterfaceDeclaration clazz) { ... }
    public static String buildRecordSignature(ClassOrInterfaceDeclaration record) { ... }
    public static String buildMethodSignature(MethodDeclaration method) { ... }
}
```

**修改范围**:
- 新建 `util/SignatureBuilder.java`
- `JavaParserAdapter.java` — 移除重复代码，使用 SignatureBuilder
- `Chunker.java` — 移除重复代码，使用 SignatureBuilder

### Config 改为 record

**问题**: Config 是唯一使用 mutable POJO 的 model，其他都是 record。

**方案**: 改为 record 类型，构建器模式：

```java
public record Config(
    Path projectRoot,
    Path dataDir,
    List<Project> projects,
    int indexingThreads,
    boolean extractJavadoc,
    // ... 其他字段
) {
    // 默认值
    public static Config defaultConfig() { ... }
    // Builder 模式
    public static class Builder { ... }
}
```

**注意**: ConfigLoader 需要适配 record 不可变性，使用 Builder 模式构建 Config。
